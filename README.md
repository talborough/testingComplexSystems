# Testing Complex Systems

## Testing Complex Systems Components

The Testing Complex Systems online experience involves 3 separate web-UIs:

- https://TestingComplexSystems.com - A readable and broad presentation of "test platforms" and a Python data handling pattern to implement them called gDS (global Data Store). There is a "publish date" listed for the site at the top of every website page.

- https://GitHub.com/talborough/testingComplexSystems (README.md) - (THE WEBSITE YOU ARE ON NOW) - A GitHub repo containing example Python test platforms and working gDS schema and table handling code. It is not version controlled like an ordinary git repo. It can be (git) cloned and viewed on either Windows or Linux-like systems but at this time it will only run on Linux. Please consult your favorite AI chatterbox for details on the issues involved. Just ask: "Python multiprocessing - Windows vs. Linux".

- https://talborough.GitHub.io/tcsDocs/ - A website of README files stripped from the above GitHub repo and presented using MkDocs and GitHub Pages. The docs are staged on https://GitHub.com/talborough/tcsDocs (Not published yet as of on 7/1/2026)
